# 스크립트 사용법 — 은둔마을 Supabase

> 모든 스크립트는 프로젝트 루트(`supabase-hermit/`)에서 실행

---

## 사전 요구사항

1. **Supabase CLI** (`npx supabase` 또는 글로벌 설치)
2. **Node.js** (npx 실행용)
3. **`.env` 파일** — 프로젝트 루트에 다음 내용:
   ```
   SUPABASE_ACCESS_TOKEN=sbp_xxxxx
   ```

---

## db.sh — Supabase CLI 래퍼

모든 DB 작업의 진입점. `.env`에서 토큰을 자동 로드하고 Supabase CLI 명령을 실행.

### 사용법

```bash
bash scripts/db.sh <command> [options]
```

### 명령어

#### `push` — 마이그레이션 적용
```bash
# 실제 적용 (성공 시 자동으로 gen-types + sync 실행)
bash scripts/db.sh push

# dry-run (변경 없이 확인만, gen-types/sync 안 함)
bash scripts/db.sh push --dry-run
```

**동작 흐름**:
1. `npx supabase db push --linked` 실행
2. 성공 시 (dry-run이 아닌 경우):
   - `gen-types.sh` 실행 -> `types/database.gen.ts` 재생성
   - `sync-to-projects.sh` 실행 -> 앱/웹 레포에 동기화

#### `pull` — Remote 스키마 가져오기
```bash
bash scripts/db.sh pull
```

Dashboard에서 직접 수정한 스키마 변경사항을 로컬 마이그레이션 파일로 가져옴.

#### `diff` — 스키마 차이 확인
```bash
bash scripts/db.sh diff
```

로컬과 Remote 스키마의 차이를 출력.

#### `lint` — RLS/스키마 린트
```bash
bash scripts/db.sh lint
```

RLS 정책 누락, 스키마 문제 등을 점검.

#### `status` — 마이그레이션 상태 확인
```bash
bash scripts/db.sh status
```

**출력 내용**:
- 로컬 마이그레이션 파일 목록
- Remote와의 동기화 상태 (dry-run 결과)

#### `gen-types` — 타입 생성 (래핑)
```bash
bash scripts/db.sh gen-types
```

`gen-types.sh`를 직접 호출하는 것과 동일.

## gen-types.sh — TypeScript 타입 생성

Supabase Remote DB 스키마에서 TypeScript 타입을 자동 생성.

### 사용법

```bash
bash scripts/gen-types.sh
```

### 동작

1. `.env`에서 `SUPABASE_ACCESS_TOKEN` 로드
2. `npx supabase gen types typescript --linked --schema public` 실행
3. 생성 결과 검증 (`export type Database` 포함 여부)
4. 헤더 주석 추가 후 `types/database.gen.ts`에 저장

### 출력 파일

```
types/database.gen.ts
```

파일 상단에 자동 생성 헤더가 포함됨:
```typescript
// This file is auto-generated by scripts/gen-types.sh
// Do not edit manually. Run 'bash scripts/gen-types.sh' to regenerate.
// Generated at: 2026-03-03T12:09:38Z
```

### 에러 케이스

- `SUPABASE_ACCESS_TOKEN` 미설정 -> 종료
- 생성된 타입에 `export type Database`가 없음 -> 종료 (Supabase 연결 문제)

## sync-to-projects.sh — 앱/웹 레포 동기화

중앙 프로젝트의 마이그레이션, 설정, 타입을 앱/웹 레포에 동기화.

### 사용법

```bash
# 앱 + 웹 모두 동기화
bash scripts/sync-to-projects.sh

# 앱만
bash scripts/sync-to-projects.sh --app

# 웹만
bash scripts/sync-to-projects.sh --web

# 변경 없이 미리보기
bash scripts/sync-to-projects.sh --dry

# 조합 가능
bash scripts/sync-to-projects.sh --app --dry
```

### 옵션

| 옵션 | 설명 |
|---|---|
| `--app` | 앱 레포만 동기화 |
| `--web` | 웹 레포만 동기화 |
| `--dry` | DRY RUN 모드 (변경 없이 확인만) |

### 동기화 대상

| 소스 (중앙) | 앱 대상 | 웹 대상 | 방식 |
|---|---|---|---|
| `supabase/migrations/*.sql` | `supabase/migrations/*.sql` | `supabase/migrations/*.sql` | 파일별 비교, 없거나 다르면 복사 |
| `supabase/config.toml` | `supabase/config.toml` | `supabase/config.toml` | diff 비교 후 복사 |
| `types/database.gen.ts` | `src/types/database.gen.ts` | `src/types/database.gen.ts` | diff 비교 후 복사 |
| `shared/constants.ts` | `src/shared/lib/constants.generated.ts` | `src/lib/constants.generated.ts` | diff 비교 후 복사 |
| `shared/types.ts` | `src/types/database.types.ts` | `src/types/database.types.ts` | diff 비교 후 복사 |

> **shared 파일 관리**: `shared/constants.ts`와 `shared/types.ts`는 중앙에서만 수정.
> 앱/웹의 기존 상수/타입 파일은 generated 파일을 re-export하는 래퍼 역할.

### 대상 레포 경로

| 레포 | 경로 |
|---|---|
| 앱 | `/mnt/c/Users/Administrator/programming/apps/gns-hermit-comm` |
| 웹 | `/home/gunny/apps/web` |

### 출력 예시

```
=== 은둔마을 Supabase 동기화 ===
중앙: /home/gunny/apps/supabase-hermit

--- [앱] /mnt/c/Users/.../gns-hermit-comm ---
  [migrations]
    ✓ 동일: 20260301000001_schema.sql
    ➕ 추가: 20260303000002_fix_group_members_recursion.sql
    결과: 1 변경, 5 동일
  [config.toml]
    ✓ 동일
  [types]
    🔄 덮어쓰기: database.gen.ts
  [shared]
    ✓ constants 동일
    🔄 덮어쓰기: database.types.ts

=== 동기화 결과 ===
중앙 migration 수: 6
앱 migration 수:  6
✅ 3 개 파일 동기화 완료
```

### 경고 사항

- 대상에만 있는 마이그레이션 파일이 있으면 경고 출력 (`중앙에 없는 migration`)
- 파일 수 불일치 시 경고 출력
- 대상 디렉터리가 없으면 자동 생성

## npm scripts

`package.json`에 정의된 편의 명령어:

| 명령 | 실행 내용 |
|---|---|
| `npm run push` | `bash scripts/db.sh push` |
| `npm run push:dry` | `bash scripts/db.sh push --dry-run` |
| `npm run pull` | `bash scripts/db.sh pull` |
| `npm run status` | `bash scripts/db.sh status` |
| `npm run lint` | `bash scripts/db.sh lint` |
| `npm run gen-types` | `bash scripts/gen-types.sh` |
| `npm run sync` | `bash scripts/sync-to-projects.sh` |
| `npm run sync:dry` | `bash scripts/sync-to-projects.sh --dry` |

## 일반적인 작업 흐름

### 새 기능 추가 (테이블/함수/정책)
```bash
# 1. 마이그레이션 SQL 작성
vi supabase/migrations/20260304000001_new_feature.sql

# 2. dry-run 확인
npm run push:dry

# 3. 적용 (자동 gen-types + sync)
npm run push

# 4. 결과 확인
npm run status
```

### Dashboard에서 스키마 변경 후
```bash
# 1. 변경 사항 가져오기
npm run pull

# 2. 타입 재생성 + 동기화
npm run gen-types
npm run sync
```

### 타입만 업데이트 하고 싶을 때
```bash
npm run gen-types
npm run sync
```

### 공유 상수/타입 수정
```bash
# 1. 중앙에서 수정
vi shared/constants.ts   # 또는 shared/types.ts

# 2. 앱/웹에 동기화
npm run sync

# 3. 앱/웹에서 re-export 확인
#    앱: src/shared/lib/constants.ts -> constants.generated.ts re-export
#    웹: src/lib/constants.ts -> constants.generated.ts re-export
#    앱: src/types/index.ts -> database.types.ts re-export
#    웹: src/types/database.ts -> database.types.ts re-export
```

## 트러블슈팅

### Windows 줄바꿈 문제 (WSL 환경)
스크립트에 `\r`이 들어가면 실행 실패:
```bash
sed -i 's/\r$//' scripts/*.sh
```

### SUPABASE_ACCESS_TOKEN 오류
`.env` 파일이 프로젝트 루트에 있는지 확인:
```bash
cat .env
# SUPABASE_ACCESS_TOKEN=sbp_xxxxx
```

### gen-types 실패 (Database 타입 없음)
Supabase 프로젝트 연결 상태 확인:
```bash
npx supabase projects list
```
