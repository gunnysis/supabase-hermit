# 맞춤형 감정 리허설 (Emotion Rehearsal) — 기획 및 설계

> 작성일: 2026-04-06 | 상태: 설계 완료 (Draft)

## 1. 배경 및 목적

은둔고립청년들은 오랜 기간 타인과의 교류가 단절되면서 '자신의 목소리를 내는 것' 자체에 대한 두려움과 생경함을 느낍니다. 텍스트 소통을 넘어 실제 사회로 복귀하기 위해서는 **목소리를 통한 감정 표현의 연습**이 필수적인 단계입니다.

본 기능은 AI를 '비심판적 청취자'로 활용하여, 사용자가 안전한 공간(방 안)에서 사회적 상황을 시뮬레이션하고 자신의 목소리와 감정을 객관적으로 확인하며 다듬을 수 있는 리허설 환경을 제공합니다.

---

## 2. 핵심 개념

### ① 감정 리허설 (Emotion Rehearsal)
- 사용자가 선택한 특정 상황(시나리오)에 맞춰 목소리로 대답하고, AI가 그 음성의 **톤, 에너지, 감정**을 분석하여 피드백을 주는 연습 과정.

### ② AI 보이스 미러링 (AI Voice Mirroring)
- "내 목소리가 남에게 어떻게 들릴까?"에 대한 답을 데이터로 제공. 
- 단순히 말을 텍스트로 바꾸는 것(STT)이 아니라, 음성 파형에 담긴 **비언어적 신호(떨림, 속도, 크기)**를 분석.

### ③ 단계적 노출 (Gradual Exposure)
- **1단계 (혼잣말)**: 오늘의 기분을 짧게 말해보기.
- **2단계 (상황 대응)**: 편의점, 면접, 전화 등 시뮬레이션.
- **3단계 (공감 연결)**: 익명으로 자신의 목소리 연습 기록을 마을에 공유 (선택 사항).

---

## 3. 상세 기능 (Features)

### 1. 시나리오 라이브러리
- 은둔 청년들이 실제 어려움을 겪는 상황 위주로 구성.
- 예: "배달 음식을 주문할 때", "오랜만에 친구에게 연락이 왔을 때", "면접에서 자기소개를 할 때".

### 2. 멀티모달 분석 피드백
- **Gemini 1.5 Pro**의 오디오 분석 기능을 활용.
- **피드백 항목**:
    - **언어적**: 답변의 적절성, 공감도.
    - **비언어적**: 목소리의 크기(자신감), 말의 속도(불안도), 어조의 안정성.
    - **격려**: "지난번보다 목소리에 힘이 실렸네요!", "따뜻한 진심이 느껴지는 톤이에요."

### 3. 감정 에너지 시각화
- 분석된 결과를 숫자가 아닌, 부드러운 파동이나 색상(Generative Art)으로 표현하여 시각적 안정감 제공.

## 4. 데이터베이스 설계 (Proposed)

### `rehearsal_scenarios` (시나리오 마스터)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `id` | SERIAL (PK) | |
| `category` | TEXT | 생활, 관계, 면접 등 |
| `title` | TEXT | 시나리오 제목 |
| `description` | TEXT | 상황 설명 |
| `ai_prompt` | TEXT | AI가 연기할 페르소나 및 질문 내용 |

### `voice_rehearsals` (사용자 연습 기록)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `id` | BIGSERIAL (PK) | |
| `user_id` | UUID (FK) | auth.users |
| `scenario_id` | INT (FK) | rehearsal_scenarios (NULL 가능 - 자유 연습) |
| `audio_url` | TEXT | Supabase Storage 경로 |
| `stt_text` | TEXT | 변환된 텍스트 |
| `analysis_result` | JSONB | {energy, stability, emotion_tags, ai_feedback} |
| `is_shared` | BOOLEAN | 마을 공유 여부 |
| `created_at` | TIMESTAMPTZ | |

## 5. 기술 아키텍처 (Technical Architecture)

1.  **Audio Record (Client)**:
    - 앱(Expo)에서 `expo-av` 라이브러리를 사용하여 고음질 녹음.
    - 웹에서 Web Audio API 활용.
2.  **Storage (Supabase)**:
    - `voice-rehearsals` 버킷에 저장 (사용자별 폴더 권한 설정).
3.  **Processing (Edge Functions)**:
    - `analyze-rehearsal` 함수 생성.
    - Supabase Storage에서 파일을 읽어 **Gemini 1.5 Pro API**로 전송 (Multimodal Prompting).
4.  **Realtime Feedback**:
    - 분석 완료 시 `voice_rehearsals` 테이블 갱신 → 클라이언트 Realtime 수신 후 결과 표시.

## 6. 유저 여정 (User Journey)

1.  **시나리오 선택**: "오늘은 편의점에서 물건 사는 연습을 해볼까요?"
2.  **AI 발화**: AI가 먼저 목소리로 말을 겁니다. "어서 오세요, 봉투 필요하세요?"
3.  **사용자 응답**: 사용자가 녹음 버튼을 누르고 대답합니다. "아뇨, 괜찮아요. 여기 카드로 계산해 주세요."
4.  **분석 및 피드백**: AI가 분석 후 리포트를 보여줍니다. 
    - *카드*: "목소리 크기가 적당하고 끝맺음이 분명해서 전달력이 아주 좋았어요!"
5.  **반복 및 성장**: 자신의 이전 기록과 비교하며 변화를 확인합니다.

## 7. 단계별 로드맵

- **Phase 1 (MVP)**: '자유 말하기' 기능. 30초 내외 음성 분석 및 감정 피드백 제공.
- **Phase 2 (Scenario)**: 시나리오 라이브러리 추가 및 대화형 UI 구현.

## 8. 기대 효과
- **사회적 근육 강화**: 실제 사람을 만나기 전 '심리적 완충 지대' 역할.
- **자기 효능감 증진**: 자신의 목소리를 긍정적으로 인식하게 됨으로써 자존감 회복.
