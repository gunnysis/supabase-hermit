## 마이페이지 종합 개선 (2026-03-18)

4차에 걸친 설계/구현. 처음부터 충분히 연구하지 않아 반복됨 — 교훈으로 기록.

### 1차: 동기화 + 안정성 (DESIGN-mypage-sync)
- **계기**: 마이페이지 "문제가 발생했습니다" 에러 신고
- **작업**: SectionErrorBoundary 9개 섹션 래핑, 로그아웃 경고 다이얼로그, 앱 EmotionCalendar 범례/스켈레톤, ActivitySummary 문구 통일
- **문제**: 실제 에러 메시지 확인 없이 추측으로 방어만 쌓음

### 2차: 에러 처리 전략 통일 (DESIGN-error-strategy)
- **계기**: "근본 원인을 해결하고 싶다"
- **작업**: API 16개 throw 표준화, QueryCache/MutationCache 전역 토스트, meta.silent, useErrorHandler 삭제
- **교훈**: 3계층 방어 (API throw → 전역 토스트 → ErrorBoundary)는 올바른 패턴이지만, 에러 메시지 먼저 확보했으면 1차에 해결 가능했음

### 3차: UX 폴리시 (DESIGN-mypage-ux-polish)
- **작업**: WeeklySummary 스켈레톤/빈 상태, EmotionWave 키보드 접근성, BlockedUsers 개별 isPending
- **문제**: 표면적 UI만 다루고 아키텍처/디자인 패턴/서비스 맥락 미검토

### 4차: 종합 설계 (DESIGN-mypage-comprehensive)
- **계기**: "아키텍처/리팩토링/디자인패턴/기능개선 등 더 연구해"
- **연구**: 서비스 정체성, 신규 사용자 경험 시뮬레이션, 네트워크 요청 분석(11개 RPC), NN/G 빈 상태 가이드, 심미적 사용성 효과
- **Phase 1 (인증 안정화)**: useAuth SIGNED_OUT → queryClient.clear() + ensureAnonymousSession (마이페이지 에러 근본 원인), signOut 에러 처리 4곳
- **Phase 2 (코드 품질)**: SectionErrorBoundary 로깅, getActivityLabel 6곳 → 중앙 1곳, EmotionTrendChart useQueries + useMemo + Set, 앱 토스트 중복 제거
- **Phase 3 (디자인 통일)**: WeeklySummary "이번 주" 문구 버그, 진행률 색상/폴백 색상/이모지/감정 칩 표준 통일
- **추가**: 웹 로그아웃 프로덕션 숨김 (앱과 동일 정책), EmotionTrendChart overflow 수정

### 교훈
1. **에러 신고 시 실제 메시지 먼저 확보** — 추측 기반 대규모 수정 금지
2. **서비스 정체성을 기능마다 확인** — 앱에서 제한한 것은 웹에서도 동일
3. **설계는 처음에 충분히** — 4번 반복하면 시간 낭비 + 문서 파편화
4. **코드 리뷰 ≠ UI 점검** — 그래프 overflow 같은 시각적 문제는 크기를 산술 계산해서 검증
5. **배포까지 확인** — git push ≠ Vercel 배포 (vercel --prod 필수)

## Daily 바텀시트 UX 개선 (2026-03-22)

- **문제**: 그림자 없음 + 블러 없음 + 단조로운 애니메이션 = "같은 평면에 떠 있는" 느낌
- **작업**: Glassmorphism 배경(BlurView + 반투명 오버레이 + border), 다층 소프트 섀도우, backdrop 50%, 핸들 40×5px stone-500, 스냅 햅틱(Light→Medium), FadeInDown 스태거드 50ms, 감정 칩 Scale 1.05, 헤더 날짜 서브텍스트
- **동기화**: 앱 배너 "다음에"→"괜찮아요" 통일, 웹 DailyPostForm 헤더 날짜 + 감정 칩 ring-2 강조
- **교훈**: 깊이감은 그림자+블러+움직임의 합. 탭바가 이미 BlurView 사용 → 동일 디자인 언어 적용으로 일관성 확보
