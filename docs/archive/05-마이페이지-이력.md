# 마이페이지 + 오늘의하루 이력

> 마이페이지 기능, 오늘의 하루(BAJ) 설계/구현 기록

---

## 마이페이지 초기 구현 (2026-03-16)

- **범위**: 프로필, 활동 요약, 나의 패턴, 감정 캘린더, 감정 타임라인
- **결과**: 웹/앱 양쪽 마이페이지 기본 구현 완료
- **교훈**: 마이페이지 섹션은 독립 컴포넌트로 분리해야 — 하나의 파일에 넣으면 유지보수 어려움

## 마이페이지 종합 점검 (2026-03-17)

- **범위**: 중앙/웹/앱 마이페이지 전체 코드 대조 (16개 이슈 식별)
- **P0**: ActivitySummary 타입 불일치, getMyAlias 중복
- **P1**: 에러 처리 누락, 불안전 캐스팅, staleTime 0, 색상 팔레트 불일치
- **P2**: ARIA 누락, 빈 상태 미처리
- **P3**: UTC 타임존, SECURITY DEFINER, DATE 캐스팅 성능
- **결과**: 16개 이슈 전체 해결, 마이그레이션 38번째 배포
- **교훈**: 공유 타입과 RPC 반환값 불일치는 로컬 타입 우회를 유발 → 중앙 타입 먼저 수정

## 웹/앱 동기화 (2026-03-17)

- **문제**: 앱에 차단 관리, 설정, 로그아웃, 개별 알림 읽음 누락
- **핵심 작업**:
  - BlockedUsersSection 컴포넌트 신규 생성 (앱)
  - useUnblockUser, useMarkRead 훅 추가 (앱)
  - 마이페이지 설정 섹션 + 로그아웃 추가 (앱)
  - DailyInsights 로딩 스켈레톤 추가 (앱)
- **결과**: 웹/앱 마이페이지 완전 동기화 (8개 섹션 일치)
- **교훈**: 웹 먼저 구현 → 앱 동기화 패턴에서 앱 구현이 누락되기 쉬움 → 체크리스트 필수

## 오늘의 하루 (BAJ) — 연구 (2026-03-15)

- **배경**: 은둔/고립 청소년의 커뮤니케이션 특성 연구 (BA/CBT 기반)
- **핵심 발견**: 긴 글쓰기 부담 → 감정+활동 태그 + 짧은 텍스트로 경량화
- **설계 원칙**: 또래 감정 공유 + 사회적 지지 → 정서적 연결

## 오늘의 하루 — 설계/구현 (2026-03-15~16)

- **범위**: posts 테이블 확장 (post_type='daily'), 전용 RPC 3개
- **핵심**: create_daily_post (원자적), update_daily_post, get_today_daily
- **결과**: Phase 0(마이페이지) + Phase 1(MVP) + Phase 2(나의 패턴) 완료
- **교훈**:
  - daily는 post_analysis를 RPC에서 직접 생성 (AI skip) — 감정분석 트리거 조건에 `post_type='post'`만
  - 하루 1회 제한은 DB에서 KST 날짜 기준으로 강제 — 클라이언트 검증만으로 부족

## 오늘의 하루 — 개선 계획 (2026-03-17)

- **범위**: 루프 완성, 프리셋 정렬, 배포 후 지표 리뷰
- **결과**: v4 종합 계획에 통합
