## 1. 컨셉

> **"오늘의 하루"는 긴 글을 쓸 힘이 없는 날에도 감정과 하루를 나눌 수 있는 가벼운 소통 포맷이다.**

```
체크인 → 피드에 공유 → 공감/리액션 → 대화 → 연결
```

버튼은 "기록하기"가 아니라 **"나누기"**. 이것은 기록이 아니라 소통이다.

| 긴 글 (post) | 오늘의 하루 (daily) |
|---|---|
| 블로그 | 트윗 |
| 제목 + 본문 필수 | 감정 칩만 필수 |
| AI 감정분석 | 사용자 직접 선택 |
| 같은 피드, 같은 리액션, 같은 댓글 ||

## 2. 설계 결정

| # | 문제 | 해결 |
|---|------|------|
| 2.1 | 신규 테이블? | **posts 확장** — `post_type` + `activities` 컬럼. 기존 인프라 전부 호환 |
| 2.2 | title/content NOT NULL | 빈 문자열 `''`. 클라이언트에서 "오늘의 하루"로 렌더링 |
| 2.3 | 감정 RPC가 post_analysis 기반 | `create_daily_post()`에서 post_analysis 직접 INSERT (status=done) |
| 2.4 | 감정분석 트리거 | 4개 트리거에 `WHEN (NEW.post_type = 'post')` 조건 추가 |
| 2.5 | 하루 1회 제한 | KST UNIQUE 인덱스 + RPC 내 범위 체크 이중 방어 |
| 2.6 | board_id | `12` (공개 게시판) 고정 |
| 2.7 | 원자적 생성/수정 | `create_daily_post()`, `update_daily_post()` SECURITY DEFINER |

**기술 스펙 참조**: 마이그레이션 `20260323000001_daily_post.sql`

## 3. 기존 시스템 호환

| 기존 기능 | 대응 |
|-----------|------|
| 뷰 (`posts_with_like_count`) | 재생성 (post_type, activities 포함) |
| 감정분석 트리거 4개 | WHEN 조건 추가 |
| 감정 RPC 4개 | **수정 없음** (post_analysis 행으로 자동 호환) |
| 검색/리액션/댓글/Realtime/RLS/soft_delete | **수정 없음** |
| 감정 캘린더/트렌드/타임라인 | **자동 포함** (post_type 필터 없음) |

## 4. UI/UX 디자인

### DailyPostCard

```
┌── bg-cream-50 / dark:bg-stone-800/50 ───┐
│  🌤️ 오늘의 하루        · 2시간 전       │
│  [😌 평온함]                             │  ← 감정 칩 (기존 표준)
│  [🚶 외출] [🍚 식사]                     │  ← 활동 태그 (outline)
│  "오늘 편의점 다녀왔다"                    │
│  ❤️ 5  💬 1                             │
└──────────────────────────────────────────┘
```

### 작성 플로우

FAB → 포맷 선택 ([📝 글쓰기] [🌤️ 오늘의 하루]) → 감정(필수) → 활동(선택) → 한마디(선택) → [나누기]

### 홈 진입점

- 미작성: "오늘은 어떤 하루예요?" [나눠볼까요?] [괜찮아요]
- 작성후: "오늘의 하루: 😌 평온함 | ❤️ 2명이 공감" [보러가기]

## 5. 개발 패턴

- **Optimistic Update**: 나누기 → 피드 즉시 반영 → 서버 확인 → 실패 시 롤백
- **캐시**: todayDaily staleTime=0, boardPosts 30초
- **컴포넌트**: PostCard에 `post_type === 'daily'` 분기 1개만 추가
- **코드 영향 최소화**: CreatePostForm 미수정, 포맷 선택 상위 배치
