## 대규모 리팩토링 실전 가이드 (2026-03-07)

- **범위**: Sentry 진단(ANR, API 에러), feature 기반 아키텍처, 성능 최적화
- **핵심 패턴**:
  - Feature 모듈: `features/{domain}/{api,hooks,components}/`
  - API 계층: `createClient()` → RPC/REST → 에러 로깅 → throw
  - 캐시 전략: TanStack Query staleTime + invalidateQueries
- **교훈**: 리팩토링은 "작동하는 코드를 더 좋게" — 기능 추가와 동시에 하면 위험

## 감정분석 구현 완벽 가이드 (2026-03-15)

- **범위**: Gemini 2.5 Flash + Structured Output + 비동기 webhook + pg_cron
- **아키텍처**: DB Trigger → Edge Function → Gemini API → post_analysis UPDATE
- **핵심 설계**:
  - 프롬프트: 한국어 전문, 청소년 맥락, 복합 감정 허용
  - 출력: emotions[] + risk_level + risk_indicators + context_notes
  - 안전장치: 60초 쿨다운, 재시도 2회, stuck 자동 정리(cleanup_stuck_analyses)
- **교훈**: LLM 호출은 실패 전제로 설계 — 쿨다운 + 재시도 + 상태 머신 + 수동 트리거 4중 안전장치
