## 관리자 시스템 재설계 (2026-03-07)

- **문제**: 그룹 게시판 관리용 관리자 → 그룹 제거 후 역할 재정립 필요
- **핵심**: 4-tab 관리자 화면 제거, long-press 히든 도어 패턴 도입
- **결과**: app_admin 테이블 유지, admin_cleanup RPC 2개 유지, UI 간소화
- **교훈**: 관리자 기능은 "운영 도구"로 한정 — 사용자 관리 기능은 별도 설계

## 관리자 앱/웹 동기화 감사 (2026-03-07)

- **범위**: 18개 기능/UX 비교 포인트 점검
- **결과**: 관리자 진입 방식 통일 (앱: AdminLongPress, 웹: AdminSecretTap)
- **교훈**: 관리자 기능은 숨겨야 하지만 접근 불가능하면 안 됨 — 히든 도어 패턴이 적절

## Sentry 에러 수정 (2026-03-08)

- **문제**: jsdom SSR 에러, DOMPurify 초기화 실패, 감정분석 파이프라인 오류
- **핵심**: isomorphic-dompurify 교체, logger/Sentry 계층 통합, PII 필터링
- **결과**: Sentry Developer 무료 플랜 유지 (2,500건/월)
- **교훈**: Sentry beforeSend에서 PII(이메일, 비밀번호) 필터링 필수 — regex로 간단 처리 가능

## EAS Build 실패 방지 (2026-03-17)

- **문제**: Expo SDK 55 업그레이드 후 companion 패키지 미업데이트로 빌드 실패
- **수정**: `npx expo install --fix` → `--check` → `expo-doctor` 3단계 필수화
- **결과**: pre-build-check.sh 스크립트에 SDK 호환성 검증 추가
- **교훈**: `npm install expo@~XX`만으로는 부족 — `npx expo install --fix`가 companion 패키지 자동 정렬

## Sentry 에러 종합 분석 + 해결 (2026-03-22)

- **범위**: 앱 24개 + 웹 4개 Sentry 이슈 전수 분석, 서비스 관점 설계 개선, 최신 기술 도입 검토
- **P0 해결**:
  - 웹 isomorphic-dompurify → dompurify 전환 (WEB-3/4: jsdom ESM 충돌, 23건)
  - block_user RPC 방어적 처리 — 별칭 미존재 시 NOOP + 자기 차단 방지 (GNS-P/Q: 20건)
- **P1 해결**:
  - extractErrorMessage 진단 강화 — 빈 에러 시 JSON.stringify로 전체 직렬화
  - blocks.ts APIError 래핑 통일 (앱/웹 모두)
  - Sentry custom fingerprint — `[API] functionName` 패턴으로 이슈 분리
- **정리**: 해결된 6개 이슈 resolved 마킹 (GNS-7/C/M/8 + WEB-1/2)
- **최신 기술 검토**: Session Replay, Structured Logs, Seer AI, SDK 8 업그레이드, offlineFirst 모드 → Phase 2 백로그
- **교훈**:
  - unknown_supabase_error(31건)는 빈 에러 객체 → JSON.stringify로 원인 추적 가능하게 해야
  - blocks.ts만 raw throw → 패턴 위반 1곳이 Sentry에서 가장 큰 노이즈 유발
  - 앱 섹션별 에러 바운더리 부재, 웹 네트워크 감지 부재 → 비대칭 해소 필요
