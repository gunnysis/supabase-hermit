# 유지보수 이력

## v1 — 종합 유지보수 (2026-03-07~09)

- **범위**: 앱/웹/중앙 3개 레포 전체 점검
- **핵심 작업**: 15개 Phase → 5개 Release로 재구성
  - R1: 보안/안정성 (npm audit, Expo 55, RLS, 보안 헤더)
  - R2: 코드 품질 (에러 처리 통일, 공유 상수/유틸)
  - R3: 문서 갱신 (SCHEMA.md, CLAUDE.md)
  - R4: DB 제약조건 (boards CHECK)
  - R5: 디자인/접근성 (MOTION 상수, prefers-reduced-motion)
- **결과**: 설계 완료 → v3 실행 계획서로 발전
- **교훈**: Phase 21개는 과부하 → Release 단위로 관리해야 실행 가능

## v2 — 서비스 개선 (2026-03-08~15)

- **범위**: Sentry 에러 수정, 감정분석 파이프라인, 검색 UX
- **핵심 작업**:
  - isomorphic-dompurify SSR 에러 해결
  - logger/Sentry 계층 통합
  - searchPosts RPC 컬럼 순서 수정
  - 감정분석 7개 구조적 결함 식별 → 쿨다운/재시도/상태추적 보강
- **결과**: 웹 Vercel 배포, 앱 테스트 통과
- **교훈**: 에러 핸들링 패턴은 처음부터 통일해야 — 나중에 수정하면 10곳 이상 건드림

## v2 기능 점검 (2026-03-16)

- **범위**: 별칭, 답글, 알림, 차단 v2 기능 전수 점검
- **핵심 작업**:
  - create_daily_post/get_today_daily 타임존 버그 (UTC→KST)
  - generate_display_alias 레이스컨디션 → EXCEPTION 재시도
  - 알림 actor_alias COALESCE 폴백
  - 차단 block_user 별칭 존재 검증
  - 웹 `as any` 7곳 제거
- **결과**: 마이그레이션 36번째 배포
- **교훈**: `CURRENT_DATE AT TIME ZONE`은 타입이 timestamp without tz → `(now() AT TIME ZONE 'Asia/Seoul')::DATE` 사용

## v3 — 종합 정비 (2026-03-16)

- **범위**: DB 제약조건 + 공유 타입/유틸 보강
- **핵심 작업**:
  - post_type CHECK 제약조건
  - 알림 페이지네이션 인덱스
  - display_alias partial unique index
  - 답글 검증 트리거 (parent_id 유효성)
  - shared/types.ts에 Notification, UserBlock, ActivityInsight 추가
  - validateDailyPostInput() 유틸 추가
- **결과**: 마이그레이션 37번째 배포, `as any` 0개 달성
- **교훈**: 타입을 중앙에서 관리하면 웹/앱 로컬 타입 우회 코드가 사라짐

## v4 — 마이페이지 종합 개선 (2026-03-17)

- **범위**: 마이페이지 16개 이슈 + DB RPC 최적화 + 웹/앱 동기화
- **핵심 작업**:
  - ActivitySummary `streak_days`→`streak` 타입 수정
  - getMyAlias() 중복 제거
  - staleTime 0→60s (웹/앱 모두)
  - SECURITY DEFINER→INVOKER 전환 (3개 RPC)
  - get_emotion_timeline KST 타임존 적용
  - get_user_emotion_calendar `::DATE`→범위 비교 최적화
  - 앱 BlockedUsersSection/로그아웃/useUnblockUser/useMarkRead 신규 구현
  - 접근성: ARIA progressbar, tooltip role 추가
- **결과**: 마이그레이션 38번째 배포, 웹/앱 마이페이지 완전 동기화
- **교훈**:
  - `staleTime: 0`은 컴포넌트 마운트마다 refetch 유발 → 최소 60s 설정
  - SECURITY DEFINER는 auth.uid() 필터링 시 불필요 → INVOKER로 충분
  - `created_at::DATE` 캐스팅은 인덱스 무효화 → 범위 비교 사용
