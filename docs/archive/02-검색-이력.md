## 검색 v1 → v2 개선 (2026-03-06)

- **문제**: ILIKE 검색의 한계 (관련도 없음, 하이라이트 없음, 감정 필터 없음)
- **핵심 작업**: search_posts_v2 RPC 설계 (풀텍스트 + 관련도 + 하이라이트 + 감정 필터)
- **결과**: 8개 개선 영역 식별, search_posts_v2 마이그레이션 작성
- **교훈**: ILIKE `%query%`는 인덱스 미활용 → ts_vector + ts_rank 조합이 정답

## 검색 리팩토링 (2026-03-07)

- **범위**: 중앙+앱+웹 검색 아키텍처 통합
- **핵심 작업**: search_posts_v2 RPC로 통합, CTE 컬럼 순서 수정, ILIKE 와일드카드 이스케이프
- **결과**: search_posts v1 제거 (deprecated)
- **교훈**: CTE 컬럼 순서가 RETURNS TABLE과 불일치하면 silent wrong data 반환 — 컴파일 에러 아님

## 검색 레이아웃 수정 (2026-03-07)

- **문제**: 감정 칩 스크롤뷰가 초기 상태에서도 표시, 감정 그리드 중복
- **수정**: 검색어 입력 전에는 감정 필터 숨김
- **교훈**: 검색 초기 화면은 최소 UI가 사용자 부담 줄임

## 검색 앱/웹 동기화 (2026-03-13)

- **범위**: 검색 페이지 디자인/UX 앱↔웹 일관성
- **핵심**: 감정 칩 카운트/스타일 통일, 감정 탐색 그리드 동기화
- **결과**: 검색 UI 앱/웹 동일 경험 달성
- **교훈**: 감정 칩은 `rounded-full px-3 py-1.5 text-xs` 표준으로 통일해야 — 컴포넌트마다 다르면 유지보수 불가
