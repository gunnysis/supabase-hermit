# 감정분석 이력

> 감정분석 파이프라인, 감정 칩 UI, 감정 타임라인 개발 기록

---

## 감정분석 재시도 시스템 (2026-03-06)

- **문제**: 비동기 webhook 파이프라인의 7개 구조적 결함 (실패 무한루프, 상태 미추적 등)
- **핵심 작업**: status/retry_count/error_reason 컬럼 추가, 60초 쿨다운, 서버 재시도 2회
- **결과**: post_analysis 상태 추적 시스템 구축
- **교훈**: 비동기 파이프라인은 "실패했는지 모르는 상태"가 가장 위험 → 명시적 상태 머신 필수

## 감정분석 실패 수정 (2026-03-15)

- **문제**: 쿨다운 버그(analyzed_at NULL 시 비교 실패), content_too_short 무한 재시도
- **수정**: analyzed_at NULL 허용, 소프트 실패 처리, stuck 분석 자동 정리 함수
- **교훈**: `now() - analyzed_at < interval '60s'`에서 analyzed_at이 NULL이면 전체 조건이 NULL → 쿨다운 우회됨

## Phase E1 — 한국어 프롬프트 + Gemini (2026-03-13)

- **범위**: 감정분석 품질 업그레이드
- **핵심**: 한국어 전문 프롬프트 + Gemini 2.5 Flash Structured Output
- **추가 출력**: risk_level, risk_indicators, context_notes (로그 기록, DB 저장은 E2 예정)
- **결과**: Edge Function 배포 완료
- **교훈**: Structured Output은 JSON 파싱 에러 제거 — `response_schema`가 타입 안전성 보장

## 감정 칩 표준화 (2026-03-13)

- **문제**: 컴포넌트마다 감정 칩 스타일 불일치 (크기, 색상, 패딩)
- **표준**: `rounded-full px-3 py-1.5 text-xs`, 활성 시 gradient[0] bg + font-semibold
- **결과**: 앱/웹 모든 감정 칩 통일
- **교훈**: 디자인 토큰은 처음부터 상수화해야 — 나중에 통일하면 20곳 이상 수정

## 감정 타임라인 개선 (2026-03-13~17)

- **문제**: 호버/탭 인터랙션 없음, 날짜 라벨 부족, 차트 높이 작음, 빈 상태 미처리
- **핵심 작업**: processEmotionTimeline() 공유 유틸 추출, 호버 툴팁, 요일 라벨, 인사이트 문장
- **결과**: 웹 EmotionWave + 앱 EmotionWaveNative 리디자인 완료
- **교훈**: 차트 데이터 변환 로직은 공유 순수 함수로 추출하면 웹/앱 중복 제거 + 테스트 용이

## 재시도 버튼 수정 (2026-03-15)

- **문제**: content_too_short 시 무한 재시도 루프, 로딩 상태 미해제
- **수정**: 소프트 실패 분기 + 최대 재시도 횟수 체크 + finally 로딩 해제
- **교훈**: 사용자 대면 재시도 버튼은 반드시 최대 횟수 제한 + 실패 사유 표시
